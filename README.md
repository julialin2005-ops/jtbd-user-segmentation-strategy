# User Segmentation Strategy Based on JTBD and Customer Journey Analysis

## What this project does

This project evaluates how effectively a product enables users to complete their Jobs-to-Be-Done (JTBD) across the full customer journey.

Instead of analyzing users statically, it identifies:
- what users are trying to achieve (JTBD)
- where the product fails across the journey
- which failures matter most commercially

The objective is to connect customer insight directly to product and commercial decisions.

## Core idea

A product succeeds or fails based on how consistently it enables users to complete their jobs at each stage of the journey.

# Customer Journey Framework

This project analyzes product performance across the full customer journey.

## Journey stages

1. Discovery & Expectation Formation  
2. Purchase & Onboarding  
3. Core Usage (Job Execution)  
4. Friction & Breakdown Points  
5. Support & Resolution  

## Key principle

A product does not fail as a whole.It fails at specific stages of the journey, where it does not successfully enable the user to complete their job.

# Jobs-to-Be-Done (JTBD) Definition

This project defines user needs as Jobs-to-Be-Done (JTBD), rather than features or preferences.

## What is a JTBD?

A JTBD describes what a user is trying to achieve in a specific situation. It is the underlying goal the user is hiring the product to accomplish.

---

## Core JTBD identified

User needs are defined as outcome-driven jobs, not features.

1. **Turn conversations into structured, usable output**  
2. **Ensure critical information is captured accurately**  
3. **Reduce effort in documenting and recalling conversations**  
4. **Fit seamlessly into existing workflows**  
5. **Validate whether the product delivers enough value to continue using**  

---

## JTBD-Based User Segments

### 1. Segment 1
**JTBD:**  
Synthesize conversations into connected intelligence to think and execute faster.  

**Key Expectation:**  
- Acts as an intelligence layer 
- Adapts and improves with usage  
- Supports multi-modal workflows  

**Risk if unmet:**  
Users build their own system or switch tools  

---

### 2. Segment 2
**JTBD:**  
Capture and convert meetings into clear decisions and actions without missing anything.  

**Key Expectation:**  
- Accurate capture  
- Clear, structured summaries  
- Share-ready output  

**Business Importance:**  
Primary driver of retention and monetization  

---

### 3. Segment 3 
**JTBD:**  
Record and retrieve information quickly with minimal effort.  

**Key Expectation:**  
- Reliability over intelligence  
- Minimal setup  
- Consistent output  


**Risk if unmet:**  
Drop-off due to complexity  

---

### 4. Segment 4
**JTBD:**  
Get from recording to usable output instantly without friction or failure.  

**Key Expectation:**  
- Zero-friction onboarding  
- Fast path from recording to output  
- No failure in critical moments  

**Risk if unmet:**  
High early churn and low activation  

---

### 5. Segment 5 
**JTBD:**  
Prove the product’s value quickly before committing to use it.  

**Key Expectation:**  
- Immediate value demonstration  
- Clear differentiation vs existing methods  
- Low commitment to try  

**Risk if unmet:**  
Low conversion despite strong product capability  

---

## From JTBD to Product Decisions

This framework enables a structured approach:

**JTBD → Expectation → Failure Point → Metric**

### Example

- **JTBD:** Never miss critical details  
- **Expectation:** Accurate capture + structured summary  
- **Failure Point:** Missing or incorrect action items  
- **Metric:** % of meetings with correctly extracted actions  

---

## Conclusion

This approach provides a practical decision-making framework to:

- Prioritize product investments based on real user impact  
- Focus on high-value segments that drive retention and revenue  
- Link user needs directly to measurable outcomes
